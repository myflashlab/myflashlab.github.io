Packagecom.myflashlab.air.extensions.gameServices
Classpublic class GameServices
InheritanceGameServices flash.events.EventDispatcher

This class is the main and the only entry point to the whole Game Services ANE. By initializing this class using the static method GameServices.init(); you will actually create a new instance of this ANE which will give you access to all the other parts of the ANE.

Make sure to take care of the permissions manually if you are targeting AIR SDK 24+: https://github.com/myflashlab/GameServices-ANE/#permissions

Method Detail
dispose()method
public static function dispose():void

Call this method to dispose the Game Services ANE altogether. After calling this method, you will need to initalize the extension again using GameServices.init(); if you wish to run the ANE again.

TIP: Normally, you won't need to dispose the extension at all, because when your game starts, the ANE will create one instance of the Game Services and you will be using only that instance through the whole life time of your app session. And when your user leaves the app, everything related to your app will be closed along with the Game Services of course.

init()method 
public static function init($isProduction:Boolean):GameServices

This is the first method you must call in your app before starting to use the Game Services ANE. This method will initialize the ANE and you need to listen to the AuthEvents.INIT event to know if Game Services has been successfully initialized or not. Based on the result status returned in the AuthEvents.INIT event, you should either call other methods of the Game Services or let the user know why she can't use Game Services based on the returned status.

Parameters

$isProduction:Boolean — set to true if you are building for production mode. However, if you are building for development, set it to false

Returns
GameServices — returns the GameServices for intense debugging reasons. you normally won't need it though.

See also

com.myflashlab.air.extensions.gameServices.google.events.AuthEvents.INIT

Examples
The following example will show you how you can initialize the extension. Please note that before you are allowed to call ANY other methods of the ANE, you must have logged in your users and you cannot login a user unless the extension has been initialized successfully. 
     import com.myflashlab.air.extensions.gameServices.GameServices;
     import com.myflashlab.air.extensions.gameServices.google.events.AuthEvents;
     
     GameServices.init(false); // set this to 'true' when you are building for production
     GameServices.google.auth.addEventListener(AuthEvents.INIT, onInit);
     
     private function onInit(e:AuthEvents):void
     {
         var result:String;
         
         switch (e.status) // status code indicating whether Game Services can run in your app or not. It can be one of following results:
         {
             case GameServices.SUCCESS:
             
                 result = "GameServices.SUCCESS";
                 
             break;
             case GameServices.SERVICE_MISSING:
                 
                 result = "GameServices.SERVICE_MISSING";
                 
             break;
             case GameServices.SERVICE_UPDATING:
                 
                 result = "GameServices.SERVICE_UPDATING";
                 
             break;
             case GameServices.SERVICE_VERSION_UPDATE_REQUIRED:
                 
                 result = "GameServices.SERVICE_VERSION_UPDATE_REQUIRED";
                 
             break;
             case GameServices.SERVICE_DISABLED:
                 
                 result = "GameServices.SERVICE_DISABLED";
                 
             break;
             case GameServices.SERVICE_INVALID:
                 
                 result = "GameServices.SERVICE_INVALID";
                 
             break;
             default:
         }
         
         trace("onInit result = " + result);
     }
