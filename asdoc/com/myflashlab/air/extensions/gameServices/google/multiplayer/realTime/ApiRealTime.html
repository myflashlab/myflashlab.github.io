createRoom()method
public function createRoom($minAutoMatchPlayers:int = -1, $maxAutoMatchPlayers:int = -1, $exclusiveBitMask:int = 0, $variant:int = -1, $addPlayersToInvite:Array = null):void

Create a real-time room for the current game. The room created by this API is a resource that needs to be released by GameServices.google.multiplayerRealTime.leave(); when the caller is done with it.

Parameters

$minAutoMatchPlayers:int (default = -1) — Minimum number of auto-matched players.
 
$maxAutoMatchPlayers:int (default = -1) — Maximum number of auto-matched players.
 
$exclusiveBitMask:int (default = 0) — Exclusive bitmasks for the automatching request. The logical AND of each pairing of automatching requests must equal zero for auto-match. If there are no exclusivity requirements for the game, this value should just be set to 0.
 
$variant:int (default = -1) — This is an optional, developer-controlled parameter describing the type of game to play, and is used for auto-matching criteria. Must be either a positive integer or -1 (the default) if not desired.
 
$addPlayersToInvite:Array (default = null) — Add a list of player IDs to invite to the room.

hideAnyNativeWindow()method 
public function hideAnyNativeWindow():void

(iOS only) hides any native window which might be open. use case: Player A invites 3 players to a game but he decides to start the game anyway as soon as Player B who is an iOS user accepts the invitation. when Player A sends the first data to player B, it should call this method to close the native window.

sendUnreliableMessage()method 
public function sendUnreliableMessage($data:String, $roomId:String, $recipientParticipantIds:Array):int

Send a message to one or more participants in a real-time room. The message delivery is not reliable and will not report status after completion. The maximum message size supported is GameServices.MAX_UNRELIABLE_MESSAGE_LEN bytes.

Leave $recipientParticipantIds parameter empty and it will send the message to all participants in the specified room.
